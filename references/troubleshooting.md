# 排错指南

## 飞书 CLI 相关

### `npm: command not found`
**原因**：未安装 Node.js。
**解决**：安装 Node.js 16+ → https://nodejs.org

### `lark-cli: command not found`
**原因**：飞书 CLI 未安装或未在 PATH 中。
**解决**：
```bash
npm install -g @larksuite/cli
# 如果仍找不到，尝试：
npx @larksuite/cli auth status
```

### 认证失败 / 401 Unauthorized
**原因**：Token 过期或凭证错误。
**解决**：
```bash
lark-cli auth login --recommend
# 浏览器重新授权
```

### 权限不足 / 403 Forbidden
**原因**：应用权限未覆盖所需操作。
**解决**：
1. 在飞书开放平台检查应用的权限配置
2. 确保包含 base:* 系列权限
3. 重新发布应用并等待审批

### Base not found / 404
**原因**：app_token 无效或 Base 已被删除。
**解决**：
1. 检查 `assets/feishu_config.json` 中的 app_token
2. 在飞书中直接打开对应的多维表格确认是否存在
3. 如已删除，重新运行初始化

## 多维表格相关

### 字段类型不支持
飞书多维表格的字段类型有限。如果遇到不支持的类型，考虑：
- 使用文本字段替代（type=1）
- 使用备注字段记录额外信息

### 记录写入失败
**常见原因**：
1. 字段名不匹配 —— 检查字段名是否与表中一致
2. 字段类型不匹配 —— 数字字段不能写入文本
3. 必填字段缺失 —— 某些字段可能是必填的

**排查方法**：
```bash
lark-cli base +field-list --base-token <app_token> --table-id <table_id>
# 检查字段名和类型是否匹配
```

### 单表记录数超限（>20,000）
**解决**：
1. 归档旧数据（导出后删除）
2. 按学期/学年分表

## 解析相关问题

### 姓名识别不准确
**原因**：自动姓名识别依赖常见姓氏库和上下文关键词。
**解决**：
1. 先录入学生基本信息，后续录入时优先匹配已知学生
2. 明确写出姓名格式：姓名：XXX
3. 如识别错误，手动修正后反馈

### 成绩提取错误
**原因**：分数格式不规范。
**支持的格式**：
- `数学95分` ✓
- `数学：95` ✓
- `语文95（第3名）` ✓
- `数学150满分考了142` △（部分支持）

**不支持的格式**：
- `95` (无从知晓科目)
- `math=95`（不支持英文科目名）

### 图片识别效果差
**原因**：手写潦草或图片模糊。
**建议**：
1. 使用清晰的照片/截图
2. 尽量使用打印版成绩单
3. 实在无法识别时手动录入

## 配置相关

### 本地配置丢失但表还在
**场景**：`assets/feishu_config.json` 被误删，但飞书多维表格仍然存在。
**解决**：
1. 在飞书中打开多维表格
2. 从 URL 中提取 app_token：`https://bytedance.feishu.cn/base/bascnXXX`
3. 在 WorkBuddy 对话中说：`我的多维表格是 bascnXXX`
4. AI 将重建本地配置

### 需要重新初始化
**解决**：
```bash
# 删除本地配置
rm assets/feishu_config.json
# 在 WorkBuddy 中说「初始化班务记录」
```
